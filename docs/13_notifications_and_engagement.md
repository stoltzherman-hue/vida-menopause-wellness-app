# Notifications and Engagement

## Principles

- Helpful, not nagging.
- User-controlled.
- Sensitive by default.
- No embarrassing lock-screen content.
- Respect quiet hours.

## Notification categories

### Daily check-in

Examples:

- "Ready for your evening check-in?"
- "Take 60 seconds to log today."

Avoid:

- "Log your menopause symptoms now" on lock screen.

### Medication/HRT reminder

Examples:

- "Medication reminder."
- "Scheduled dose due."

Avoid medication names on lock screen unless user opts in.

### Refill reminder

- "You may be running low soon."

### Insight ready

- "Your weekly pattern summary is ready."

### Community

- Replies, Circle updates, supportive reactions.
- Allow digest mode.

### Doctor prep

- Appointment report reminder.
- User enters appointment date.

### Wellness plan

- Weekly plan check-in.
- Strength training reminder.
- Sleep routine reminder.

## Quiet hours

Default:

- 21:00-07:00 local time.

Exception:

- User can opt into bedtime wind-down prompts.

## Engagement loops

1. Check-in -> insight -> personalized suggestion -> next check-in.
2. Community post -> reply -> Circle suggestion -> return.
3. Symptom pattern -> doctor report -> appointment prep -> updated plan.
4. Medication reminder -> adherence -> trend summary -> refill reminder.
5. Sleep issue -> wind-down companion -> sleep check-in next morning.

## Metrics

- Notification opt-in rate.
- Check-in completion from reminder.
- Disable rate.
- Community return rate.
- Weekly active tracking days.
- Premium conversion after insight notification.

## Safety

- Never send crisis-like content as automated notification.
- Do not reveal sensitive sexual/genitourinary topics on lock screen.
- Do not use fear-based reminders.
